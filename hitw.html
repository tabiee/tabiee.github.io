Overview


An open-world adventure game that was a passion project for a group of students from the year before mine. This game started out as their university project and later continued into further development and eventually a company and dedicated production.

I came into the picture after the game has already released a demo and was seeking help to continue developing it, meaning I had no part in the initial idea or structure of the project. I was unfamiliar with the team members, meeting each for the first time.

What I learned

My involvement in the project included occasional visits to the office space the team occupied and providing assistance as a programmer and tester for the game. I primarily took care of bugfixing, refactoring, adding small game mechanics and various testing.

What I learned:


  • Working in an on-going project
  • Getting familiar with someone else's code
  • Refactoring a godclass
  • Using the Command pattern
  • Constructing test cases, like test trees and test rooms
  • Finding playtesters and writing a feedback form
  • Creating a basic save system with JSON, Playerprefs and NewtonSoft.Json
  • Daily Scrum
  • Bugtracking and reporting
  • Collaborating on programming and testing with other people
  • Using GitHub branches
  • Making Steam Achievements
